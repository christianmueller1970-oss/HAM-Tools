## Kurz-Überblick

- **Activator / Hunter**-Modi auswählbar beim Anlegen einer POTA-Session
- **Park-Auto-Complete** aus der mitgelieferten POTA-Park-Datenbank (lokale SQLite)
- **Multi-Park-Hopping**: mehrere Park-Referenzen pro QSO (Komma-Liste)
- **POTA-Spots-Feed**: Live-Stream von api.pota.app statt regulärer DX-Cluster-Spots
- **POTA-Map**-Tab: Park-Pins + DX-Pins + Linien Park→DX
- **POTA-Stats** im Awards-Tab: Activator-Parks, Hunter-Parks, P2P-QSOs
- **ADIF-Konformität**: `MY_POTA_REF`, `POTA_REF`, `MY_SIG`, `MY_SIG_INFO`

## Anlegen einer POTA-Session

1. Top-Bar → **Neues Log** → **POTA-Session** → öffnet den POTA-Wizard
2. **Activator** oder **Hunter** wählen
3. **Eigener Park** eintragen (Activator) — Auto-Complete sucht in der Park-DB
4. Optional: **Hopping-Parks** als Komma-Liste (`K-1234, K-5678`)
5. **Anlegen** → Log ist erstellt, POTA-Form wird aktiv
